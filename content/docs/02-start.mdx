This guide will help you set up your development environment and get started with the Fit Schedule Planner project.

## Prerequisites

Before you begin, make sure you have the following installed:

- [Node.js](https://nodejs.org/) (Latest LTS version)  
- [MongoDB](https://www.mongodb.com/) (Community Edition)  
- [Git](https://git-scm.com/)  
- A code editor (VS Code recommended)  

## Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/moura-pedro/fit-schedule-planner.git
   cd fit-schedule-planner
2. **Install frontend dependencies:**
   ```bash
   cd client
   npm install
   ```

3. **Install backend dependencies:**
   ```bash
   cd ../server
   npm install
   ```

4. **Set up environment variables:**  
   Create a `.env` file in the `server/` directory with the following:
   ```bash
   MONGODB_URI=your_mongodb_connection_string
   JWT_SECRET=your_jwt_secret
   OPENAI_API_KEY=your_openai_api_key
   ```

## Running the Application

1. **Start the backend server:**
   ```bash
   cd server
   npm start
   ```
   The server will run on port 3000 by default.

2. **Start the frontend dev server:**
   ```bash
   cd client
   npm start
   ```
   The React dev server will run on port 3001.

## Project Structure

### Frontend (`client/`)
- `src/components/` – Reusable React components  
- `src/pages/` – Page components  
- `src/hooks/` – Custom React hooks  
- `src/utils/` – Utility functions  
- `src/styles/` – Global styles & Tailwind config  

### Backend (`server/`)
- `routes/` – API route definitions  
- `controllers/` – Route handlers  
- `models/` – Mongoose models  
- `services/` – Business logic  
- `helpers/` – Utility functions

## Common Issues

### Port Conflicts  
If you encounter port conflicts:  
- Ensure ports 3000 and 3001 aren’t in use  
- You can change them in your server’s & client’s config files  

### MongoDB Connection Issues  
- Verify MongoDB is running  
- Double‑check your connection string in `.env`  
- Inspect MongoDB logs for errors  

### Node Module Issues  
If node modules misbehave:
```bash
rm -rf node_modules
rm package-lock.json
npm install
```
